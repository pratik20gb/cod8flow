## Project Overview

**cod8flow** is a task-management backend API (think Jira/Trello) built with Spring Boot 4.x, Java 21, PostgreSQL, and Redis. It is a phased learning project — phases 1–3 (setup, JWT auth, core domain) are complete; phase 4 (Redis caching + S3) is in progress.

## Commands

```bash
# Build (skip tests)
./mvnw clean package -DskipTests

# Run (requires PostgreSQL on :5433 and Redis on :6379)
./mvnw spring-boot:run

# Run all tests
./mvnw test

# Run a single test class
./mvnw test -Dtest=SomeTestClass

# Compile only
./mvnw compile
```

> On Windows use `mvnw.cmd` instead of `./mvnw`.

## Infrastructure Requirements

Two services must be running locally before the app starts:

| Service    | Default port | Config key                     |
|------------|-------------|--------------------------------|
| PostgreSQL | **5433**    | `spring.datasource.url`        |
| Redis      | 6379        | `spring.data.redis.host/port`  |

Override DB credentials via env vars `DB_USERNAME` / `DB_PASSWORD` (defaults: `cod8flow`/`cod8flow`).

## Architecture

### Layer Structure

```
controller  →  service  →  repository (Spring Data JPA)
                ↓
           domain/entity  (JPA entities, all use UUID PKs)
           domain/enums   (Role, WorkspaceRole, TaskStatus, Priority)
           dto/request    (validated input)
           dto/response   (API output, never expose entities directly)
           security/      (JwtService, JwtAuthFilter, UserDetailsServiceImpl)
           config/        (SecurityConfig, RedisConfig)
```

### Domain Model

```
User  1──* WorkspaceMember *──1  Workspace  1──* Board  1──* Task
                                                         Task *──1 User (assignee)
```

- All PKs are `UUID` generated by Hibernate (`GenerationType.UUID`).
- `Workspace` owns `Board` and `WorkspaceMember` via `CascadeType.ALL + orphanRemoval`.
- `Board` owns `Task` the same way.

### Authentication Flow

1. `POST /api/v1/auth/register` or `/login` → returns `accessToken` + `refreshToken`.
2. `JwtAuthFilter` intercepts every request, validates the Bearer token via `JwtService`, and sets the `SecurityContext`.
3. `POST /api/v1/auth/refresh` exchanges a valid refresh token for a new access token.
4. JWT secret is hex-encoded in `application.yml`; `JwtService` decodes it with `hexToBytes()`.
5. Stateless sessions (`SessionCreationPolicy.STATELESS`).

### Public Endpoints

`/api/v1/auth/**`, `/api/v1/health`, `/actuator/**` — no JWT required.

### Database Migrations

Flyway manages all DDL. Migration files live in `src/main/resources/db/migration/` named `V{n}__{description}.sql`. **Never edit an existing migration file** — always create the next version. The schema is validated at startup (`ddl-auto: validate`).

### Redis / Caching (Phase 4, in progress)

`RedisConfig` exposes a `RedisCacheConfiguration` `@Bean` with a 10-minute TTL and JSON value serialization via `GenericJacksonJsonRedisSerializer`.

**Jackson 3 note:** Spring Boot 4.x / Spring Data Redis 4.x use **Jackson 3**, whose package is `tools.jackson.databind.*` (Jackson 2 was `com.fasterxml.jackson.databind.*`). The serializer's builder manages its own mapper — there is no `.objectMapper(...)` method. Enable polymorphic typing with `.enableDefaultTyping(PolymorphicTypeValidator)` using a Jackson 3 `BasicPolymorphicTypeValidator` (`allowIfBaseType(Object.class)`). Jackson 3 auto-registers the Java Time module, so `LocalDateTime`/`Instant` fields serialize without manually adding `JavaTimeModule`.

Use `@Cacheable` / `@CacheEvict` on service methods. Note that `allowIfBaseType(Object.class)` enables permissive default typing — acceptable for caching our own DTOs, but not for untrusted data.

### Current User Resolution

Services retrieve the authenticated user from `SecurityContextHolder` via `SecurityContextHolder.getContext().getAuthentication().getName()` (which returns the email), then load the full `User` entity from the DB.

### Lombok Usage

All entities and most DTOs use Lombok (`@Builder`, `@Getter`, `@Setter`, `@RequiredArgsConstructor`). The annotation processor is wired in `pom.xml` for both compile and test-compile phases.

## API Route Summary

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/auth/register` | Register new user |
| POST | `/api/v1/auth/login` | Login |
| POST | `/api/v1/auth/refresh` | Refresh access token |
| GET/POST/DELETE | `/api/v1/workspaces` | Workspace CRUD |
| GET/POST/DELETE | `/api/v1/workspaces/{id}/boards` | Board CRUD |
| GET/POST/PATCH/DELETE | `/api/v1/boards/{boardId}/tasks` | Task CRUD |
